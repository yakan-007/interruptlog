## 所見

### P1-01: 記録シートを閉じると、割り込み／休憩が見えないまま継続する

- 再現手順
  1. タスクを開始する。
  2. 割り込みまたは休憩を開始する。
  3. 記録シートの `閉じる`、または背景をタップする。
- 期待結果
  - 閉じるがキャンセルの意味なら、直前のタスクへ復帰し、未保存の一時停止を破棄する。
  - 継続させる設計なら、閉じる前に「記録を中断して継続する」ことを明示して確認する。
- 実際の結果
  - シートだけが閉じ、`running` は `interrupt` または `break` のまま残る。タスクは停止済みで、時間は一時停止として積算され続ける。
  - 実行バーをタップすれば再び開けるが、利用者が閉じる操作を取り消しと受け取ると、後続の作業が正しく記録されない。
- 影響範囲: タスク、履歴、日／週／月／年レポート、CSV、バックアップ。
- 原因: `App.jsx` の `openSheet` が先に一時停止を開始する一方、`SheetShell` の閉じる／スクリーン操作が UI を閉じるだけで `cancelPauseInState` を呼ばない。
- 改善案: 割り込み・休憩のシート専用 `onClose` を設け、未保存ならキャンセルして元タスクを再開する。継続を許す場合は、明示的な「あとで記録」操作を別に置く。
- 再確認条件: 閉じる／背景タップ／Esc 相当の全経路で、元タスクが再開され、履歴とレポートの合計に未保存の一時停止が増えない。

### P2-01: 開始直後にタスク累計が `-1m` と表示される

- 再現手順
  1. 空の状態からタスクを `追加して開始` する。
  2. 1 秒の ticker 更新前にタスクカードの累計を確認する。
- 期待結果: 開始直後の累計は `0m`。負の時間は表示しない。
- 実際の結果: `累計 -1m` が一瞬表示された。
- 影響範囲: 実行中タスクカード。記録値ではなく表示だが、時間計測アプリとして信頼感を損ねる。
- 原因: `TaskCard` が ticker の古い `now` と新しい `runningStart` の差をそのまま `fmtDurationMin` に渡し、`Math.floor(-0.x)` が `-1` になる。
- 改善案: `liveSpent` を `Math.max(0, now - runningStart)` でクランプする。秒表示も同じ不変条件にそろえる。
- 再確認条件: 開始・再開・別タスクへの切替直後を連続で操作しても、`-` を含む経過時間が一度も表示されない。

### P2-02: 「キャンセル」が 5 秒超の割り込み／休憩を履歴へ記録する

- 再現手順
  1. タスクを開始し、割り込みを開く。
  2. 5 秒以上待つ。
  3. `キャンセル` を押す。
  4. 履歴または日レポートを開く。
- 期待結果: 表示名がキャンセルなら履歴に残らないか、時間を残す仕様なら「記録せず再開」ではなく保存相当の文言で予測可能にする。
- 実際の結果: `キャンセルした割り込み` が割り込み時間として履歴・レポートに残る。
- 影響範囲: ユーザー理解、割り込み統計、CSV。
- 原因: `cancelPauseInState` が 5 秒超の一時停止をイベント化する一方、UI 文言が破棄を意味する `キャンセル` である。
- 改善案: 挙動を維持するなら `記録して再開` などに改名する。破棄を意図するならイベントを作らない。安全性の観点では、保存する設計と明示的な「破棄して再開」を併記するのが望ましい。
- 再確認条件: 5 秒未満／以上の両方で、ボタン名・確認メッセージ・履歴結果が一致する。

### P3-01: 今日の「押し忘れ」初期値が未来の終了時刻になる

- 再現手順
  1. 今日の履歴で `押し忘れ` を開く。
  2. 初期表示の開始・終了時刻を確認する。
- 期待結果: 「押し忘れ」は現在より未来の記録を初期入力しない。
- 実際の結果: 開始を現在時刻、終了をその 30 分後として初期化するため、未来の時間帯になる。実行中タスクがある場合は、開始が現在分に丸められるため重複エラーにもなりやすい。
- 影響範囲: 履歴入力。データ破損ではないが、初期値のまま保存すると未来のイベントを作れる。
- 原因: `createDefaultMissedDraft` が今日の終了時刻を `now` でなく日末に対してクランプしている。
- 改善案: 今日では終了を `now` 以下にし、開始を `now - 30分`（日初を下限）にする。未来記録を明示的に許す要件があるなら、画面名と説明を「手動イベント追加」に変更する。
- 再確認条件: 今日・過去日・日末付近で初期値が有効な 1 分以上の範囲となり、今日分は現在時刻を超えない。

## 回帰テスト提案

最小限の自動 UI 回帰ケースとして、既存の `src/App.smoke.test.jsx` に次を追加する。

1. 割り込みと休憩を開き、閉じる／背景タップ後に元タスクが再開され、未保存イベントが残らないこと。
2. `追加して開始`、復元して開始、タスク切替の直後に、累計・経過時間が 0 以上であること。
3. 5 秒超の一時停止で「キャンセル」を選んだ結果を、採用した仕様どおりに履歴とレポートで確認すること。
4. 今日の押し忘れを初期値のまま保存しようとしても、未来時刻を作らないこと。
5. JSON の正常復元・不正復元・全削除の後、再読み込みしてローカル保存状態が期待どおりであること。
