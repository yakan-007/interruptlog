# ロードマップ: Web v1 完成 → iPhone アプリ

最終更新: 2026-07-03

## ゴール定義

- **Web v1 完成** = 「個人向け・日本語UI・ローカル完結」のスコープで、不具合ゼロ・データ消失対策あり・公開体裁ありの状態。ここで一区切り。
- **iPhone アプリ完成** = Web の移植ではなく、「iPhone に住んでいるからこそ割り込みを正確に記録できる」体験が成立している状態。

## Part 2: iPhone アプリの完成形

### コンセプト

「割り込みが発生する場所(通知・電話・Slack が鳴るデバイス)と同じ場所に記録係が住んでいる」こと。
Web 版との違いは機能の多さではなく、**記録までの距離がゼロになる**こと。

### 技術路線

- Capacitor で現行 React コードをラップ + ネイティブ機能(Swift)をプラグインで追加
- 状態管理・ロジック・テストは Web と共有し続ける
- サーバーは持たない(買い切り IAP は StoreKit 2 でデバイス内完結)

### 「これができてこそ iPhone アプリ」の定義(優先順)

1. **Live Activities / Dynamic Island** — 計測中タスク名と経過時間がロック画面・Dynamic Island に常時表示。
   割り込み発生時、アプリを開かずロック画面から「中断」をタップできる。
   → タイムトラッカーとして Web に絶対できない体験。審査ガイドライン 4.2(Web ラップ拒否)対策の柱でもある。
2. **ロック画面・ホーム画面ウィジェット** — 今日のタスク時間 / 割り込み回数の表示、タップで即記録。
3. **App Intents(ショートカット / Siri)** — 「割り込み記録」「タスク開始」を Siri・ショートカット・アクションボタンから実行。
4. **ローカル通知** — 計測しっぱなし検知(例: 2時間超)、朝の開始リマインド。
5. **ネイティブ作法の細部** — ハプティクス(開始/停止時)、スワイプジェスチャー、Face ID なしの即起動、オフライン完全動作(元々ローカル完結なので自然に満たせる)。

### やらないこと(iOS の制約上不可能 or スコープ外)

- 電話・通知の自動検知(iOS のプライバシー制約で不可能。構想から外す)
- クラウド同期・アカウント(将来 iCloud/CloudKit 同期は検討余地があるが v1 では持たない)
- iPad / Mac Catalyst 対応(iPhone に集中)

### マネタイズ

- フリーミアム + 買い切り Pro(1,000〜1,500円、StoreKit 2)
- 無料: 記録・履歴・日次振り返り(習慣が付く部分)
- Pro: 週/月/年分析、割り込み分析(発信者別・時間帯別・復帰時間)、日報/週報出力、CSV/JSON、ウィジェット複数種
- 無料/有料の線引きは**初回リリース時に決めて明示**(後からの有料化はレビューが荒れる)
- App Store Small Business Program 申請(手数料 30%→15%)
- Apple Developer Program 年 99 ドルが唯一の固定費

### マイルストーン

1. **M1: 実機で動く**(〜2週間)— Capacitor 導入、手元の iPhone で全機能動作。無料 Apple ID で可、Program 登録不要
2. **M2: iPhone らしさ第1弾**(1〜1.5ヶ月)— Live Activities + ウィジェット1種 + ハプティクス。Developer Program 登録
3. **M3: 無料版として審査提出**(M2 完了後すぐ)— プライバシーポリシー(Part 1-D を再利用)、スクリーンショット、App Store 文言。まず審査を通す経験を積む
4. **M4: Pro 追加**(+1〜1.5ヶ月)— 割り込み分析の拡充 + StoreKit 2 買い切り IAP + Small Business Program
5. **M5: 広げる**(以降)— App Intents、通知、英語ローカライズ + ASO

**iPhone アプリ「完成」= M4 終了時点。** M5 は完成後の成長フェーズ。

## Part 3: 構想メモ — 現場チーム版(作業者→現場監督の日報集約)

個人版とは別プロダクトの構想。個人版 v1 / iPhone 版の完成を優先し、これは検証から始める。

### コンセプト

- 作業者: 各自のスマホで InterruptLog に記録し、終業時に日報データを出力して送る(LINE / メール / 共有フォルダ)
- 現場監督: 集約ツールに複数人のファイルを読み込み、日報の確認・チームの時間分析(誰がどの作業に何時間、割り込み・手待ちの発生状況)を行う

### サーバーレスで成立させる MVP 案

- 作業者側: 既存アプリに「日報ファイル出力(1日分の JSON)」を追加するだけ
- 監督側: **別のローカル完結 Web アプリ**(集約ビューア)。複数 JSON をドラッグ&ドロップ → チーム日報表・人別/作業別集計・CSV/印刷出力
- アカウント・サーバー・同期なしで検証可能。既存の state ロジックを流用できる

### マネタイズの考え方

- 払うのは作業者ではなく**監督(会社)**。B2B は単価が個人アプリの10倍以上取れる(月数千円/現場でも成立)
- MVP 段階は集約ビューアの買い切り or 現場単位ライセンスで様子見
- 本格化(クラウド収集・アカウント)はニーズ検証後。サーバーを持った瞬間に責任と運用コストが質的に変わるため、**検証前に作らない**

### 検証済みの事実(2026-07 時点)

- 自分の職場では「メモ紙に時間と作業内容を書く → Excel に手打ち転記」が現状フロー。
  = 課題は実在し、自分自身が最初のユーザー兼検証現場になれる

### 最初の一歩

- [ ] 自分の職場の Excel フォーマットを確認し、集約ビューアの出力目標にする(**現行 Excel と同じ列構成で出力できること**が置き換えの条件。xlsx 直接出力 or Excel に貼れる形式)
- [ ] 転記作業に毎日何分かかっているか計る(→ 効果の数字として営業・値付けの根拠になる)
- [ ] 集約ビューアのプロトタイプ(1〜2週間規模)を作り、まず自分の記録で回す → 同僚 1〜2 人に広げる
- [ ] 回り始めたら他現場・他社の監督にも見せて、汎用ニーズと価格感を確認
